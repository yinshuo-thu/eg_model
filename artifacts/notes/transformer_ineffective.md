# Transformer v2 — ineffective ideas (dead-ends)

- Causal EMA temporal smoothing of predictions (per-instrument, alpha 0.7/0.5/0.3/0.2) — test IC 0.0566/0.0511/0.0415/0.0347 IR 0.767/0.669/0.519/0.418 — monotonically WORSE than baseline (0.05906/0.846). The forward return has very low day-to-day persistence, so mixing in past-day predictions dilutes today's signal. Smoothing is out.
- Post-hoc per-day prediction transforms on baseline preds: winsorize (3/2.5/2 std), per-day rank, per-day gaussianize — all LOWER test IC and IR (winsor3: 0.0576/0.817; rank: 0.0502/0.731; gauss: 0.0553/0.794). Baseline preds are already well-conditioned; squashing removes signal. Out.
- Weight EMA decay 0.999 + cosine-to-zero LR (3 seeds) — test IC 0.05649 IR 0.71 (valid 0.05780/0.76) — worse. EMA too slow vs cosine schedule: the averaged weights lag the optimum so per-seed IC falls ~0.005 below baseline. Slow EMA is out (try faster decay instead).
- Larger lookback K=48 (4 seeds) — test IC 0.05778 IR 0.856 (valid 0.06071/0.88) — lower test IC than K=32 (0.05895). K=64 not run. Longer windows dilute the short-horizon signal; only marginal diversity value in a blend.
- Group-g embedding conditioning (GEMB, add learned per-group embedding to input proj; 6 seeds, K32) — test IC 0.05772 IR 0.76 (valid 0.06172/0.84) — lower test IC AND IR than plain. The model fits a group tilt that doesn't generalize to test; net negative. Out as a primary, marginal diversity only.
- Cross-sectional IC loss, day-batched (DAYBATCH=1, ICW=3.0, ~4 days/batch) — valid IC stuck ~0.040 through ep7 (vs ~0.058 clean) — destabilizes training; the small day-batch + strong corr term gives noisy gradients. Killed early. Out.
- Diverse-blend IC ceiling: grid-searching weights over {b8, v2B, k48, gemb} to maximize VALID IC caps base TEST IC at 0.05904 (best weights b8:2, gemb:0.5). All transformer variants share the same signal + valid/test gap; blending cannot push base test IC above ~baseline. Neutralization then can only trade that IC down for IR — so neutralization-alone cannot clear (IC>=0.0591 & IR>1.10). Need a genuine test-IC gain or much higher base IR.
- "Stable-tilt-preserving" neutralization: remove only the daily DEVIATION of each group's tilt from its historical (<=day880) average, keeping the persistent group tilt. WORSE IC/IR slope than plain neutralization (deviation a=1.0: test 0.05607/1.027 vs plain a=0.5: 0.05850/1.007). The persistent group tilt carries ~zero test IC, so keeping it just adds non-predictive cross-sectional structure. Plain alpha*groupmean neutralization remains the most IC-efficient stability lever (slope ~ -0.0062 test IC per +1.0 IR).
- Factor/PCA neutralization: residualize pred against top-K (5/10/20) global feature PCs per day. Strongly WORSE (K10 a=0.5: test 0.05297/0.817; a=1.0: 0.04154/0.702) — the feature principal components ARE the alpha, so removing them destroys IC. Out.
- Cross-sectional (cross-instrument) attention: per-day, all instruments attend to each other after the temporal encoder (run_transformer_v3.py, NCS=2, 3 seeds). VALID IC 0.06293 IR 0.96 but TEST IC 0.05708 IR 0.93 — OVERFITS. Higher valid, lower test; the model exploits same-day cross-sectional structure that does not generalize to the 881+ test regime. Capacity is not the bottleneck — the valid->test gap is. Out. (Also confirms diverse architectures don't lift base test IC above ~0.0590.)
- Diagnostic: clean ≤760 runs peak VALID IC at epoch ~6 (range 4-8) then overfit; the valid->test IC gap (~0.063 vs ~0.059) is the binding wall for clearing the gate.
